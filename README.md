# Training Availability Tool

A two-part tool for collecting and analyzing training availability from your team.

## Overview

This tool consists of two Streamlit applications:

1. **Availability Submission App** - For team members to submit their availability
2. **Availability Planner** - For organizers to analyze and find the best training sessions

## Setup

### Install Dependencies

```bash
pip install -r requirements.txt
```

## Usage

### Part 1: Collecting Availability

Run the submission app for your team to fill in their availability:

```bash
streamlit run availability_submission_app.py
```

**Features:**
- 🔒 Password protected access
- Simple web form for team members to enter their name and availability
- Four availability options: Available, Tentative, Prefer not, Unavailable
- Responses are automatically saved to `availability_responses.csv`
- Team members can update their responses by submitting again with the same name
- Shows a summary of current responses
- Admin can download all responses as CSV

**Default Password:** `training2026` (⚠️ Change this before deploying!)

**Customization:**
Edit the `TRAINING_SESSIONS` list in `availability_submission_app.py` to match your training dates:

```python
TRAINING_SESSIONS = [
    "Mon 20 Jul AM",
    "Mon 20 Jul PM",
    "Tue 21 Jul AM",
    # Add more sessions...
]
```

### Part 2: Analyzing Availability

Once you have collected responses, run the planner app:

```bash
streamlit run training_availability_planner.py
```

**Features:**
- Upload the CSV file with all responses
- See ranked sessions by availability
- View weighted scores for each session
- Download analysis results as Excel
- See detailed breakdown of who can attend each session

## Workflow

1. **Setup**: Customize the training sessions in `availability_submission_app.py`
2. **Collect**: Share the submission app URL with your team
3. **Download**: Once everyone has responded, download the CSV from the submission app
4. **Analyze**: Upload the CSV to the planner app to find the best sessions
5. **Decide**: Use the rankings and weighted scores to schedule your training

## Data Format

Both apps work with a simple CSV format:

```csv
Name,Mon 20 Jul AM,Mon 20 Jul PM,Tue 21 Jul AM
Joe,Available,Tentative,Unavailable
Kevan,Unavailable,Available,Available
```

## Tips

- **For small teams**: Have everyone submit through the app, then download the CSV
- **For large teams**: Deploy the submission app online (Streamlit Cloud, Heroku, etc.)
- **Multiple training events**: Use different data files or folders for each event
- **Privacy**: The CSV only stores names and availability status

## Availability Options

- **Available**: Person can definitely attend
- **Tentative**: Person might be able to attend (counts as 50% in weighted scoring)
- **Prefer not**: Person prefers not to attend (counts as 25% in weighted scoring)
- **Unavailable**: Person cannot attend

## Security

⚠️ **Important Security Notes:**

**Current Protection:**
- Password-protected (default: `training2026`)
- Change password in `.streamlit/secrets.toml`

**Security Options:**

1. **Password Only (Recommended for small teams):**
   - Simple and effective
   - Just share password with your team
   - Good for 5-50 trusted people

2. **Password + Email Restrictions (Optional for extra security):**
   - Use Streamlit Cloud email restrictions
   - Requires Google sign-in
   - Good for large organizations or external partners

**Risks:**
- Anyone with the password can submit responses
- Password could be shared widely
- All authenticated users see response count

**Best Practice:**
- Share password only via internal channels (email, Slack)
- Change password regularly if needed
- Download CSV regularly as backup
